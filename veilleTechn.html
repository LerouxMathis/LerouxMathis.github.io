CVE-2023-0567

Une vulnérabilité de sécurité a été découverte dans les versions antérieures de PHP 8.0.X, 8.1.X et 8.2.X, dans la fonction "password_verify()" qui vérifie si un mot de passe correspond à un hachage stocké dans une base de données. En raison d'un bug dans la fonction, elle peut accepter certains hachages de Blowfish invalides comme valides, ce qui peut permettre à des utilisateurs malveillants de se connecter à l'application avec n'importe quel mot de passe. Il est recommandé de mettre à jour vers une version plus récente pour corriger cette faille de sécurité. La dernière mise à jour de cette information a été effectuée le 10 mars 2023.

CVE-2023-0662

Une vulnérabilité de sécurité dans les anciennes versions de PHP (8.0.x, 8.1.x et 8.2.x) permet à un attaquant de provoquer un déni de service en soumettant un nombre excessif de parties dans un formulaire HTTP d'upload de fichiers. Bien que la vulnérabilité soit notée 0.0 dans le système de notation CVSS, elle peut causer des problèmes de sécurité si elle est exploitée. La mise à jour vers les versions récentes de PHP (8.0.28, 8.1.16 ou 8.2.3) est recommandée pour corriger cette vulnérabilité.
